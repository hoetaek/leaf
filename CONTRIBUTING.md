# Contributing

Thanks for considering a contribution to `leaf`.

## Development Setup

```bash
git clone https://github.com/hoetaek/leaf.git
cd leaf
cargo test --locked --all-features
```

## Checks

Run these before opening a pull request:

```bash
cargo fmt --all --check
cargo check --locked --all-targets --all-features
cargo clippy --all-targets --all-features -- -D warnings
cargo test --locked --all-features
```

If you change dependencies, make sure `Cargo.lock` is updated.
License/source/bans policy is checked in CI with `cargo deny`.

## Versioning

`leaf` follows pre-1.0 SemVer.

- Patch: bug fixes and internal changes
- Minor: new features, scaffold layout changes, and breaking CLI/state
  changes while the tool is still pre-1.0
- Major: breaking changes after 1.0.0

The version is managed in `Cargo.toml`.

## Release

Releases are generated by cargo-dist from `dist-workspace.toml`.

1. Update `Cargo.toml` and `CHANGELOG.md`.
2. Push the release commit to `main`.
3. Push a SemVer tag such as `v0.2.0`.

The release workflow publishes GitHub Release artifacts and updates
`hoetaek/homebrew-tap`. The repository secret `HOMEBREW_TAP_TOKEN` must have
contents read/write access to that tap.

## Pull Requests

Keep changes focused. Include tests for behavior changes, especially when
touching command parsing, slug validation, git repository detection, or
scaffold layout.
