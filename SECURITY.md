# Security Policy

## Supported Versions

Security fixes target the latest released version.

## Reporting a Vulnerability

Please do not open a public issue for suspected vulnerabilities.

Use GitHub's private vulnerability reporting for this repository if available.
If it is not available, contact the maintainer through the GitHub repository
owner profile and include enough detail to reproduce the issue.

Useful details include:

- affected version or commit
- operating system
- command and config involved
- expected and actual behavior
- any sensitive local files or credentials that may be exposed

Do not include real credentials in reports.
